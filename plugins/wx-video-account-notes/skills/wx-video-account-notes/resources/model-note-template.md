# 模型成稿模板

## 成稿原则

- 只基于已有材料整理，不要编造内容
- 优先参考 `note_materials.json`
- 必要时对照 `raw.json`、`ocr.txt`、`asr.txt`
- 如果 `note_materials.json` 包含 `visual_frames` 字段，且当前模型有识图能力，应读取这些图片理解画面内容，用图片信息补足 OCR / ASR 无法表达的版式、界面、图表、人物动作或视觉上下文
- 保留"提取到的文案"和"转写文案"的区别
- 输出应自然、可读，不要堆叠原始识别结果

## 推荐结构

```md
# 标题

- 原始链接：
- 作者：
- 视频文件或图片素材：

## 内容梗概

把原来的内容概览、剧情梗概、关键信息合并成一节。
用 4-8 条自然中文，交代主题、冲突、关键推进和结果。

## 提取到的文案

这部分是融合 OCR、ASR、元数据后的整理稿，用来提炼画面文字、口播信息和上下文。
由模型自然分段，不要机械编号成“第 1 段、第 2 段”，也不要按 ASR 原顺序逐句抄写。

## 转写文案

这部分是以 ASR 为主、结合明显纠错后的清洗转写稿，尽量保留原始表达顺序和细节，便于后续继续改稿。
可以整理口语赘余和明显误识别，但不要把它改写成上一节那种融合整理后的概述正文。

## 提取状态

- OCR：
- ASR：

## 局限说明

说明哪些内容来自 OCR / ASR / 元数据 / visual_frames，以及可能存在的误识别或画面理解不确定性。
```

## 约束提醒

- 这个 skill 不预设固定业务场景，不要因为当前样例像影视解说，就对文本做垂类专用纠错
- 遇到存疑词句时，宁可保留不确定性，也不要强行替换成你以为更合理的词
